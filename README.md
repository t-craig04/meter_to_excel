###Summary###

Taking meter readings are common 'busy-work' tasks for many technicians, often which are in outside conditions, have different angles, and would require mannally inputting data into sheets, either through a phone application, writing it down somewhere then transfering it to a computer, or some other way where someone needs to spend time going to the meter and gathering their information themselves. There currently exists software that does this for automatic meter readings, but this is ideal for a real, end-to-end engineering system. Ideally I would like to use this for Forest Lawn Outdoor Pool meter readings so I can ~~play clash royale~~ spend more time with other tasks, and ensure that I am well rested for guarding. 

In this project, there will be minimal AI-generated code, as one of the objectives is for me to build an end-to-end system and be able to talk about my process, methodology, challenges and be able to handle the implementation portion of this project, on my own. As a recent B.Sc. graduate in Data Science, I am highly concerned of the reliance of AI for assignments, projects, and a lot of software-engineering repositories from peers studying computer science, and lack of understanding what is really happening. Therefore, this also serves as a learning opportunity to learn about git, github, proper coding practices, the virtual environment, and future problems that would occur in an environment like this.
